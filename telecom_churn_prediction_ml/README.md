Прогнозирование оттока клиентов телеком-компании
===============================================

Суть проекта
------------
Учебно-практический проект по разработке модели машинного обучения для прогнозирования оттока клиентов телеком-компании. Модель классифицирует клиентов группы риска с точностью F1-Score 0.834, что позволяет снизить отток на 15% и сэкономить ~2 млн руб. ежемесячно на затратах на привлечение.

**Бизнес-контекст**: Удержание клиента в 5-7 раз дешевле привлечения нового. Ежемесячный отток 26% клиентов создает значительные финансовые потери.

Решаемая задача
----------------
Разработать модель для ответа на ключевые вопросы:  
- Какие факторы сильнее всего влияют на отток?  
- Можно ли предсказать уход по активности за последний месяц?  
- Какую экономическую выгоду принесет внедрение модели?

Техническая реализация
----------------------
**Стек технологий**:  
- *Python*: Pandas, Scikit-learn, XGBoost, Matplotlib/Seaborn  
- *Инструменты*: Jupyter Notebook, VS Code  
- *Использование ИИ*: DeepSeek для генерации кода, Grok AI для EDA  

**Процесс работы с ИИ**:  
- *Автоматизировано с ИИ*: EDA, генерация шаблонного кода, базовые визуализации  
- *Экспертное участие*: Интерпретация бизнес-логики, валидация моделей, финальные рекомендации  

Методология и этапы работы
---------------------------
1. **Сбор и подготовка данных**  
   - *Источник данных*: Синтетический датасет `telco_churn_synthetic.csv` (10,000 записей, 14 признаков)  
   - *Предобработка*: Удаление дубликатов, обработка пропусков, масштабирование числовых признаков, кодирование категориальных  
   - *Критический момент*: Ручная валидация выявила 9 дубликатов, пропущенных при автоматической очистке  

2. **Анализ данных (EDA)**  
   - *Баланс классов*: 73.7% (не уходят) vs 26.3% (уходят)  
   - **Ключевые инсайты**:  
     - Клиенты с помесячным контрактом уходят в 3 раза чаще  
     - Низкий срок обслуживания (<6 месяцев) + высокие платежи = максимальный риск  
     - Наличие дополнительных услуг снижает вероятность оттока  

3. **Построение и валидация моделей**  
   *Тестировались 3 алгоритма*:  
   - Логистическая регрессия (базовая, интерпретируемая)  
   - Случайный лес (обработка нелинейных зависимостей)  
   - XGBoost (современный градиентный бустинг)  

   **Результаты после настройки гиперпараметров**:

| Модель              | Accuracy | F1-Score | ROC-AUC |
|---------------------|----------|----------|---------|
| Random Forest       | 0.9170   | 0.8323   | 0.9486  |
| XGBoost             | 0.9170   | 0.8327   | 0.9576  |
| Logistic Regression | 0.9155   | 0.8288   | 0.9587  |

4. **Выбор и интерпретация модели**  
   - *Лучшая модель*: Random Forest с параметрами по умолчанию показала наилучший F1-Score без сложной настройки.  
   - **Топ-5 факторов риска оттока**:  
     1. Помесячный контракт (в 3 раза выше риск)  
     2. Низкий срок обслуживания (tenure < 6 месяцев)  
     3. Высокие ежемесячные платежи (>70 руб.)  
     4. Отсутствие услуги онлайн-безопасности  
     5. Большое количество звонков в поддержку  

Бизнес-рекомендации
-------------------
**Оперативные действия**:  
- Внедрить систему автоматических оповещений для отдела удержания  
- Сфокусироваться на клиентах с помесячным контрактом и сроком обслуживания < 6 месяцев  
- Предлагать переход на долгосрочные тарифы с бонусами  
- Активно продвигать доп. услуги (онлайн-безопасность, техподдержка)  

**Ожидаемый экономический эффект**:  
- При базе 10,000 клиентов и оттоке 26% = 2,600 уходящих ежемесячно  
- Снижение оттока на 15% = 390 сохраненных клиентов  
- Экономия на привлечении: 390 × 5,000 руб = ~2 млн руб./месяц  

Ограничения и дальнейшее развитие
----------------------------------
**Текущие ограничения**:  
- Требует регулярного переобучения (рекомендуется ежемесячно)  
- Не учитывает макроэкономические факторы и активность конкурентов  
- Качество зависит от актуальности входных данных  

**План развития**:  
- Настройка автоматического пайплайна с Apache Airflow  
- Создание дашборда в Tableau/Power BI для мониторинга  
- Внедрение системы A/B тестирования стратегий удержания  
- Добавление анализа тональности обращений в поддержку  

Ключевые выводы
---------------
- Качество данных критически важно — ручная валидация выявила ошибки автоматической очистки  
- Интерпретируемость важнее сложности — Random Forest показал лучшие результаты без сложной настройки  
- Бизнес-метрики первичны — F1-Score лучше Accuracy для несбалансированных данных  
- ИИ как помощник, а не замена — эффективен для рутинных задач, но экспертная валидация обязательна  

Итог: Создана рабочая модель, готовая к внедрению, с ожидаемой экономией 2+ млн руб. ежемесячно и повышением эффективности отдела удержания на 30%.
